# TravelUP

- User Stories: 

As a developer, I want to extend the data model so that a Quote can include hotels with name, cost per night, 
and total cost, as well as flights with departure and arrival airports, flight number, dates, and cost, 
so that the system reflects detailed travel options. Adapt existing screens accordingly. (X)

- User Stories: 

As a developer, I want to update the data model to include city and country information, and create screens and logic to upload cities and countries from a CSV file, 
so that location data is comprehensive and manageable. (X)

- User Stories:

As a developer, I want to adapt existing screens to use city and country data from the database, and implement dropdowns to select first the country and then the city, so that location selection is user-friendly and accurate. (X)

- User Stories:
As a manager, Mary wants to create, edit, and import projects for travelers to use when creating requests, 
so that project data is managed effectively. Add a projects table to the database and update screens accordingly.

- User Stories:
As a facilitator, Frank wants to view submitted or ongoing travel requests prioritized by due date, so that he can efficiently manage urgent requests.

- User Stories:
As a manager, Mary wants full control over approving travel requests, including cases where costs exceed budget, so that she can make informed approval decisions.

- User Stories:
As a developer, I want to add authentication to the app and create users in the database, so that only authorized users can access the app screens and logic.

- User Stories:
As a developer, I want to create roles (Traveler, Facilitator, Manager) and assign them to users using Role Based Access Control (RBAC), so that access to screens is appropriately restricted or granted.
